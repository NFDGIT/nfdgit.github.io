网络、存储、路由、性能、调试、工程与 pub。原理与基础见 [面试-原理](面试-原理.md)。

## 一、网络与存储

**常用网络库与区别**

- **http**：官方轻量库，够用。**Dio**：拦截器、取消、FormData、下载进度等，企业项目常用。选型看是否需要拦截器、重试、全局配置。

**本地存储方案**

- **shared_preferences**：K-V，适合配置、简单缓存。**sqflite**：SQLite，结构化、复杂查询。**path_provider + 文件**：大文件、自定义格式。**hive / isar**：NoSQL 本地库，高性能、可选加密。

**如何安全存 token？**

- 敏感数据用 **flutter_secure_storage**（走 Keychain/Keystore），不放在 SharedPreferences 明文。

## 二、路由

**Navigator 1.0 与 2.0**

- 1.0：命令式 push/pop，简单。2.0：声明式，路由栈由 state 驱动，适合深链接、权限控制、复杂流程。

**命名路由与传参**

- 在 MaterialApp.routes 里注册名字；pushNamed 传参可用 arguments 或 query。复杂对象建议用 RouteSettings.arguments。

**go_router 使用场景**

- 声明式、path 与 state 对应、支持 deep link、嵌套路由；中大型项目常用。

## 三、性能

**如何排查卡顿（jank）？**

- DevTools Performance 看帧时间、找出 build/layout/paint 过重的帧；Profile 模式看 CPU。常见原因：build 里重计算、大列表不用 builder、过多 RepaintBoundary 或滥用。

**ListView 优化**

- 用 ListView.builder 懒加载；避免 item 里过重 build；长列表考虑 list_view_item_builder 或分页。

**Overdraw 是什么？如何减少？**

- 同一像素被画多次；用 DevTools 的「Show overdraw」看；减少不必要的层叠、透明背景、RepaintBoundary 隔离静态区域。

**const 与性能**

- const Widget 可复用、减少 rebuild 时的比较与创建；能加就加，尤其列表 item。

## 四、调试与 DevTools

**Flutter DevTools 能做什么？**

- Inspector 看 Widget/Element 树、布局边界；Performance 看帧、CPU；Memory 看内存；Network 看请求；Debug 设置断点、热重载。

**热重载与热重启区别**

- 热重载：注入代码、保留状态、只重建受影响子树。热重启：重启 isolate，状态清空，适合改 initState 或全局变量。

**Release 与 Debug 差异**

- Release：AOT、去掉断言与调试信息、优化多；Debug：JIT、支持热重载、DevTools、性能差一些。

## 五、工程与 pub

**pubspec.yaml 常用字段**

- dependencies / dev_dependencies：依赖。flutter 的 assets、fonts。environment：Dart/Flutter SDK 约束。

**依赖冲突怎么解决？**

- `flutter pub upgrade` 看可升级版本；`dependency_overrides` 慎用；优先改自己依赖版本或找兼容的包。

**多环境配置（dev/staging/prod）**

- 用 flavor 或环境变量区分；不同 entry、不同 API 地址与配置；可用 --dart-define 或 .env 文件。

**插件（Plugin）与包（Package）区别**

- Plugin：含 Dart + 平台代码（iOS/Android），封装设备能力。Package：可纯 Dart，只做逻辑或 UI 组件。

## 六、综合与场景

**Platform Channel 使用注意**

- 异步、不阻塞 UI；大数据考虑分片或换方式；异常要在原生侧捕获并返回，避免崩溃。

**混合开发里 Flutter 与原生如何通信**

- 通过 MethodChannel/EventChannel；原生嵌 Flutter 时由 FlutterEngine 持有 channel，两边约定 name 与编解码方式。

**如何做国际化？**

- 用 flutter_localizations + intl 或 easy_localization；文案放 arb 或 json，按 locale 加载；日期、数字用 Intl 格式化。

**单元测试与 Widget 测试**

- 单元测试测逻辑、不依赖 Flutter 框架。Widget 测试用 testWidgets + Finder 找 Widget、模拟点击与状态。集成测试用 integration_test 跑真机/模拟器。

**如何减少包体积？**

- 用 --split-debug-info、--obfuscate；去掉未用资源与字体；检查依赖、去掉重复或过大的包；Android 用 App Bundle。

**Flutter 与原生混编时内存谁管？**

- Flutter Engine 与 Dart VM 由 Flutter 管；原生侧由原生管。嵌 Flutter 的页面退出时要释放 Engine，避免泄漏。

**简述 Flutter 的渲染流程（一帧）**

- BuildOwner 处理 dirty Element → build；PipelineOwner 处理 dirty RenderObject → layout → paint → 产出 Layer；合成、光栅化、上屏；等 VSync。

**如何做灰度或 A/B 下发？**

- 从服务端或本地配置读策略；用不同 Widget 分支或不同数据源渲染；可结合路由、环境变量或 Feature Flag 库。

**Flutter Web 与移动端差异**

- 渲染可走 CanvasKit 或 HTML；路由与 deep link 要兼顾 URL；包体积与首屏更敏感；部分插件仅支持移动端，需查兼容性。

**如何做埋点与监控？**

- 用 Platform Channel 调原生埋点 SDK，或 Dart 侧发 HTTP 到自家日志服务；关键流程打点、异常上报；Release 包可带混淆，注意符号表备份便于还原堆栈。

**Flutter 如何适配多屏与折叠屏？**

- 用 MediaQuery、LayoutBuilder 做响应式布局；折叠屏关注 `DisplayFeature`、窗口大小变化；测试时用不同分辨率与折叠状态。

**如何做无障碍（Accessibility）？**

- 用 Semantics、MergeSemantics、ExcludeSemantics 标注语义；给图标/按钮加 semanticLabel；测试用 TalkBack/VoiceOver 与 DevTools 的 Accessibility 检查。

**Flutter 与原生页面如何互相打开与传参？**

- 原生开 Flutter：启动 FlutterEngine、打开 FlutterViewController/Activity，通过 MethodChannel 或 initialRoute 传参。Flutter 开原生：MethodChannel 调原生「打开某页」，原生再回传结果到 Dart。

**dart:io 与 dart:html 在跨平台时的处理**

- 条件导入：`import 'xxx_stub.dart' if (dart.library.io) 'xxx_io.dart' if (dart.library.html) 'xxx_web.dart'`；各平台实现不同文件，避免在 Web 里引 io、在移动端引 html。

**如何做灰度发布或按需加载模块？**

- 服务端下发开关或版本；Dart 侧根据开关决定是否注册路由、是否加载某模块；可结合 deferred loading 延迟加载 Dart 库，减小首包。

**Flutter 里如何做大列表无限滚动？**

- ListView.builder +  scrollController.listen，滑到底部时加载下一页；配合 paging 库或自己维护 page 与 loading 状态；注意避免重复请求、节流。

**如何做主题与暗色模式？**

- ThemeData 配亮/暗两套；用 `Theme.of(context).brightness` 或 MediaQuery.platformBrightnessOf 判断；切换时更新 MaterialApp.theme / darkTheme 或整体 state，整 app 重建即可生效。

**Flutter 里 JSON 序列化最佳实践**

- 手写 toJson/fromJson 或代码生成：json_serializable + build_runner；复杂嵌套用 freezed 等；避免运行时反射带来的体积与兼容问题。

**Widget 测试里如何 mock 依赖？**

- 用 Provider/Repository 注入 mock 实现；或 Override 某 provider（Riverpod）；testWidgets 里 pumpWidget 时包一层提供 mock 的 InheritedWidget/ProviderScope。

## 工程与排错（补充）

**Release 包崩溃如何定位？**

- 保留符号表或上传 dSYM（iOS）/ Proguard mapping（Android）；用 Crashlytics、Sentry 等收堆栈；Flutter 侧用 `FlutterError.onError`、`runZonedGuarded` 抓未捕获异常；原生侧看 native 堆栈是否在引擎或插件。

**如何减小 Flutter 包体积？**

- 用 `--split-debug-info`、`--obfuscate` 减小体积并混淆；去掉未用资源与字体；Android 打 App Bundle 按 ABI 分包；分析 `flutter build apk --analyze-size` / 使用 DevTools 的 size 工具查大对象。

**多 Flutter 引擎（多实例）要注意什么？**

- 每个引擎独立 isolate、独立内存与插件注册；注意路由、Navigator 与 Overlay 是 per-engine 的；共享状态需通过 MethodChannel 或自己维护；避免单页多引擎时的重复注册与泄漏。

**Flutter 与原生内存如何隔离？**

- Flutter 引擎跑在独立线程、有自己堆；Dart VM 与原生通过 Platform Channel 通信，不共享堆；大对象传参用 callback 或文件/共享内存，避免 channel 传巨大数据。

**CI 里如何跑 Flutter 测试与 build？**

- 安装 Flutter SDK 并 `flutter pub get`；`flutter test` 跑单测；`flutter analyze` 做静态检查；build 用 `flutter build apk/ios --release`，可配合 codemagic、Codemagic、GitHub Actions 等；缓存 `~/.pub-cache` 与 Flutter 目录加速。
