# Flutter 面试 · 原理与基础

原理类、Dart、Widget 生命周期、Key、状态管理。实践与工程见 [面试-实践](面试-实践.md)。

---

## 一、基础与概念

**StatelessWidget vs StatefulWidget**

- **StatelessWidget**：无状态、不可变，适合静态或由父组件驱动的 UI。
- **StatefulWidget**：带 `State`，通过 `setState()` 触发重建，用于会变的 UI。

**BuildContext 是什么？**

- 表示 Widget 在 **Element 树**里的「位置」；用来拿 Theme、Navigator、找父/子、Overlay 等，不是「当前 Widget 的引用」。

**const 构造函数**

- 编译期常量，减少重复创建、利于 Flutter 复用 Element，能加就加。

**MaterialApp vs WidgetsApp**

- MaterialApp：自带 Material 设计、主题、路由、本地化等。
- WidgetsApp：更底层，只提供基础导航与标题，不绑 Material/Cupertino。

## 二、三棵树与渲染

**Widget / Element / RenderObject 三者关系？**

- Widget：配置单（轻量、可大量建）。Element：挂载点，对应 Widget 实例、管更新。RenderObject：真正做 layout/paint 的，存几何信息。见 [底层原理](底层原理.md)。

**为什么 Widget 要不可变？**

- 便于对比与复用 Element；声明式「每次 build 返回完整描述」，框架负责 diff 与更新。

**约束（Constraints）如何传递？**

- 自上而下传约束，自下而上回报尺寸；父根据子的 size 排位置。紧约束/松约束/无界的区别要能说清。

**Layer 与 RepaintBoundary 的作用？**

- Layer 是「可复用的绘制块」，合成时叠在一起；RepaintBoundary 让子树画到独立 Layer，重绘时只重画这块，减少 Overdraw。

## 三、Dart 语言

**Event Loop 与 Isolate**

- Dart 单线程 + Event Loop；microtask 先于 event queue。Isolate 之间内存隔离，靠 SendPort/ReceivePort 消息通信。

**async/await 与 Future**

- async 函数返回 Future；await 挂起当前协程不阻塞 isolate，等 Future 完成再继续。要区分「同步代码 / microtask / event」执行顺序。

**空安全（Null Safety）**

- 类型默认非空，可加 `?` 表示可空；`!` 断言非空；`late` 延迟初始化。避免 null 误用、编译期就能发现很多问题。

**mixin 与继承**

- mixin 混入能力而不必单继承；可多个 mixin，解决「多继承」需求。常见如 `SingleTickerProviderStateMixin`。

**final、const、late 区别**

- final：单次赋值；const：编译期常量；late：延迟初始化，第一次访问时赋值。

## 四、Widget 生命周期与 Key

**StatefulWidget 生命周期**

- createState → initState → didChangeDependencies → build → (重复) setState → build → deactivate → dispose。initState 里做一次性初始化；dispose 里取消订阅、释放资源。

**initState 与 didChangeDependencies 区别**

- initState 只调一次；didChangeDependencies 在 initState 后以及依赖的 InheritedWidget 变化时调用，可在这里依赖 context 做逻辑。

**Key 的作用与种类**

- Key 帮助 Element 在兄弟顺序变化时决定复用还是新建。LocalKey（ValueKey、ObjectKey）同父下唯一；GlobalKey 全局唯一，可跨节点、拿 State 或 Find Widget。

**什么时候必须用 Key？**

- 列表重排（如 reorderable list）、动态增删子 Widget 且要保留状态时；同类型兄弟若没有 Key，Flutter 按位置复用可能错乱。

## 五、状态管理

**为什么需要状态管理？**

- 跨组件共享状态、避免层层传参；业务逻辑与 UI 解耦、易测试；数据流清晰、可预测。

**Provider / Riverpod / Bloc 选型**

- Provider：简单、依赖 Context，中小项目。Riverpod：不依赖 Context、编译期安全、异步友好，中大型首选。Bloc：事件→状态、逻辑与 UI 彻底分离，企业级、样板多。

**InheritedWidget 与 Provider 关系**

- Provider 底层就是 InheritedWidget + 状态封装；依赖者会在数据变时重建。

**Bloc 的 Event 与 State**

- 用户/系统产生 Event，Bloc 里转成新 State；UI 只读 State 并 dispatch Event，单向数据流，易测试与回溯。

**热重载（Hot Reload）原理**

- 把改动的 Dart 代码注入 VM，保留应用状态，只重建受影响子树；改 initState 或静态变量往往需热重启才生效。

**BuildContext 使用注意**

- 在 build 里拿的 context 只在当前帧有效；异步回调里用 context 要检查 mounted，避免 disposed 后访问。

**Widget 与 Element 的对应关系**

- 一个 Widget 可对应多个 Element（同一 Widget 被多处使用）；一个 Element 对应一个 Widget 和最多一个 RenderObject（若该 Widget 是 RenderObjectWidget）。

**setState 后一定重建整棵子树吗？**

- 只重建当前 State 对应的 Element 及其子树的 build；若子 Widget 没变（同类型、同 Key），子 Element 可复用、不重建。

**InheritedWidget 更新时谁会重建？**

- 只有通过 dependOnInheritedWidgetOfExactType（或 watch）依赖过它的 Element 会标记 dirty 并重建；只 read 不 depend 的不会。

**Dart 的 mixin 与 interface 区别**

- mixin 混入实现；interface 只约定接口。Dart 3 的 interface 可带默认实现，与 mixin 的用法有重叠，需区分「继承」与「混入」语义。

**GlobalKey 的典型用法**

- 跨层获取 State（如 Form 里拿子 FormField 状态）、在非 build 阶段拿 context、保证某 Widget 在树里唯一对应一个 State。慎用：破坏局部性、易误用。

**为什么说「Widget 是配置、Element 是实例」？**

- Widget 可被多次创建、可相等可不等；Element 是树上真实存在的节点，和 RenderObject/State 一一对应，负责「挂载」和「更新」。

**dependOnInheritedWidgetOfExactType 与 inheritFromWidgetOfExactType 区别**

- depend：建立依赖，InheritedWidget 更新时当前会标记 dirty 并重建。inherit：只读一次，不建立依赖，不随其更新重建。需要「跟着变」用 depend，只取一次用 inherit。

**Stream 与 Future 在状态管理中的用法**

- Future：一次性异步结果，FutureProvider 或 FutureBuilder。Stream：持续数据流，StreamProvider 或 StreamBuilder；Bloc 内部也用 Stream 传 state。

**Cubit 与 Bloc 区别**

- Cubit 无 Event，直接调方法改 state；Bloc 收 Event、在 mapEventToState 里转成 state。Cubit 更轻，Bloc 更「事件溯源」、适合复杂流程。

**Flutter 的「声明式」具体指什么？**

- UI 由「当前状态」唯一决定；开发者只描述「给定状态下的界面长什么样」，不直接操作 DOM 或 View；状态变 → 重新 build → 框架 diff 后只更新变化部分。

**BuildContext 在异步回调里为什么可能失效？**

- Widget 可能已被 dispose，对应 Element 从树移除；异步回来时再拿 context 操作（如 push、showDialog）会抛异常。解决：用前检查 `context.mounted`，或把 context 存为可空、回调里再判空。
