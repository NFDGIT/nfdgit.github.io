# Flutter 调试与排错

DevTools 使用、常见崩溃与排查、Release 符号化。言简意赅。

---

## DevTools 常用能力

| 工具 | 用途 |
|------|------|
| **Widget  inspector** | 点选页面看对应 Widget/Element/RenderObject 树，查布局与层级。 |
| **Performance** | 录帧、看 timeline，定位 build/layout/paint 谁耗时长、是否掉帧。 |
| **Memory** | 看堆快照、查泄漏；对比操作前后快照找未释放对象。 |
| **Network** | 看 Dart 侧发起的 HTTP 请求（需配合插件或 DevTools 版本）。 |
| **Show repaint bounds** | 在真机/模拟器上显示重绘边界，判断 RepaintBoundary 是否合理。 |

- **一句话**：布局问题看 inspector、卡顿看 Performance、内存涨看 Memory、重绘看 repaint bounds。

## 常见崩溃与排查

- **「setState() called after dispose()」**：异步回调里对已销毁的 State 调 setState。解决：回调里先 `if (!mounted) return;` 再 setState，或取消订阅（如 StreamSubscription.cancel、Timer.cancel）。
- **「Looking up a deactivated widget's ancestor」**：在 Widget 已从树移除后仍用其 context（如 push、showDialog）。解决：用前 `context.mounted` 判断，或把操作放到 postFrameCallback 里并再判 mounted。
- **Release 包闪退、堆栈是乱码**：未保留符号或未混淆对应。解决：build 时加 `--split-debug-info=dir`、`--obfuscate`，把生成的符号上传到 Sentry/Crashlytics，用其后台还原堆栈。
- **原生层崩溃（引擎或插件）**：看设备日志（adb logcat / Xcode 控制台），堆栈里找 Flutter 引擎或插件 so/framework；插件问题查插件源码与 issue，引擎问题查 Flutter 版本与已知 bug。
- **一句话**：Dart 侧先看 mounted 与 dispose 时机；Release 必留符号；原生崩看原生堆栈与插件/引擎版本。

## 日志与断言

- **开发期**：`debugPrint`、`log()` 不截断；`assert` 在 Debug 下生效，Release 被去掉。
- **生产期**：避免打印敏感信息；需要可打采样日志或通过 FlutterError.onError / runZonedGuarded 统一上报。
- **一句话**：开发用 assert 与 debugPrint，Release 用上报替代大段日志。
