## Context

站点大部分页面已完成现代化（主题、动画、响应式），但相册、工具、赛车和笔记搜索仍有明显短板。本次是跨模块横向治理，每个子项独立性强。

## Goals / Non-Goals

**Goals:**

- 相册 3D 立方体改为 flexbox 居中 + 视口单位缩放，移动端自动缩小。
- 工具页添加 3 个纯前端工具（JSON 格式化、颜色转换、二维码），卡片式交互。
- 赛车 UI 颜色适配暗色主题。
- 笔记侧边栏添加实时搜索过滤。

**Non-Goals:**

- 不替换相册的 3D 立方体为其他布局（保留创意）。
- 不重写赛车游戏引擎。
- 不做服务端搜索。

## Decisions

- **相册居中方案**：用 `display: flex; justify-content: center; align-items: center; min-height: 100vh` 替代 `margin-left: 42%; margin-top: 22%`。3D 立方体用 `vmin` 单位适配不同屏幕。
  - 备选：改为 CSS Grid。
  - 取舍：flex 居中更简单且只有一个元素，采用。

- **工具页架构**：每个工具是一个 `<details>` 可展开卡片，内含输入区 + 输出区 + 操作按钮。JS 逻辑写在 `tools/tools.js` 中。
  - 备选：每个工具独立页面。
  - 取舍：工具都很轻量，单页更方便切换，采用。

- **二维码生成**：使用 `qrcodejs` CDN（~5KB），纯前端生成 Canvas 二维码。
  - 备选：手写 QR 编码。
  - 取舍：不必要的复杂度，采用成熟库。

- **赛车主题适配**：不修改 Canvas 内的绘制颜色（那是游戏视觉设计），只修改 UI 层（开始画面背景、文字、按钮）的颜色为 CSS 变量读取。
  - 备选：全量改颜色。
  - 取舍：Canvas 内改动风险大且意义不大，只改 UI 层。

- **笔记搜索**：在侧边栏 header 下方添加 `<input>` 搜索框，用 `oninput` 实时过滤 `manifest` 中的文件名（模糊匹配），隐藏不匹配的树节点。
  - 备选：全文内容搜索。
  - 取舍：全文搜索需要加载所有文件内容，性能不可控，先做文件名搜索。

## Risks / Trade-offs

- **[风险]** 赛车 `app.js` 是第三方代码，修改可能引入 bug。
  **缓解：** 只改 UI 初始化阶段的颜色常量，不动游戏循环逻辑。

- **[风险]** 二维码库 CDN 不可达。
  **缓解：** 二维码工具在库加载失败时显示"功能不可用"提示。

- **[风险]** 相册 3D 效果在低端设备卡顿。
  **缓解：** 不改变 3D 动画本身，只改定位方式。
