# Flutter 笔记目录 — 设计

## Context

站点 `note/` 下已有 iOS（含 README、interview、collect 等）、IT、AI 等目录，manifest 由脚本扫描生成。用户要求新建 Flutter 目录，覆盖底层原理、面试、高阶用法等，内容从网络获取并对比择优，语言生动、言简意赅。

## Goals / Non-Goals

**Goals:**

- 建立 `note/Flutter/`，与现有笔记结构一致，便于左树导航。
- 按「底层原理」「面试」「高阶用法」等分类组织，每类至少一个文档，可读性强。
- 内容基于网络资料对比整理，优先官方文档与高赞/高星资料，避免过时或错误信息。
- 文风：生动形象、言简意赅，适当使用小标题、列表、表格与简短示例。

**Non-Goals:**

- 不替代 `note/IT/flutter_fvm.md`，可在 Flutter README 中链入。
- 不实现自动化爬取或定时同步；实现阶段由人工/AI 基于检索结果撰写与校对。

## Decisions

1. **目录与文件结构**  
   - 根级：`note/Flutter/README.md`（索引 + 到各子文档的链接）。  
   - 主题文档：扁平放置于 `note/Flutter/` 下，便于与 iOS 风格一致、manifest 简单。  
   - 建议文件：`底层原理.md`、`面试.md`、`高阶用法.md`；若单文件过长，可再拆为多文件（如 `面试-基础.md`、`面试-原理.md`）或子目录，由实现时按篇幅决定。

2. **内容来源与择优**  
   - 底层原理：Flutter 官方架构文档、Dart 文档、渲染管线与 Widget 体系文章；GitHub 上的原理梳理。  
   - 面试：GitHub 上的 Flutter 面试题仓库、掘金/知乎等高赞总结、大厂面经。  
   - 高阶用法：官方进阶指南、状态管理（Provider/Riverpod/Bloc）对比、性能与平台通道最佳实践。  
   - 对比时优先：官方 > 高星/高赞 > 时效性；同一知识点取表述更清晰、更简练的一版，合并成文。

3. **语言与格式**  
   - 生动形象：适当比喻、图示说明（可用 ASCII 或 Mermaid）、「一句话记住」式小结。  
   - 言简意赅：避免冗长复述，用列表与表格替代长段落；代码示例短小精悍。  
   - 中文为主，术语保留英文（如 Widget、BuildContext、setState）。

## Risks / Trade-offs

- **[取舍]** 网络内容会更新，笔记可能滞后 → 在 README 或文末注明「以官方文档与最新资料为准」，重要结论附来源或链接。
- **[取舍]** 一次实现难以覆盖所有面试题与进阶点 → 先成主干（底层原理、面试、高阶用法各一篇），后续变更可增量补充。
