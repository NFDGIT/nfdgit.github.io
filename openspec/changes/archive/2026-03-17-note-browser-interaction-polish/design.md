## Context

笔记页采用左右分栏布局（侧边栏 + 预览区），桌面端侧边栏可收起，移动端用抽屉式。当前交互的核心问题是：导航/控制按钮散落在侧边栏 header 和预览区 topbar 两处，收起侧边栏后控制按钮消失，移动端浮动按钮与内容冲突。

## Goals / Non-Goals

**Goals:**

- 添加一个始终可见的顶部工具条（`.glass-nav` 风格），集中所有导航和控制功能。
- 桌面端收起侧边栏 = 侧边栏完全消失（width: 0），不留 48px 窄条。
- 移动端不再有独立浮动的目录按钮，统一由工具条中的按钮触发抽屉。
- 面包屑从预览区移入工具条。
- 移除预览区内的 topbar（返回首页 + 面包屑），让预览区从内容开始。

**Non-Goals:**

- 不改变笔记搜索、高亮、树节点图标等功能。
- 不改变移动端抽屉的滑入动画和背景模糊。

## Decisions

- **工具条布局**：左侧放目录切换按钮 + 返回首页；中间放面包屑（flex: 1，溢出截断）；右侧放主题切换和大字模式。工具条使用 `.glass-nav` 但改为 `justify-content: space-between`（而非 `flex-end`），因为需要左中右三区。

```
┌────────────────────────────────────────────────────┐
│ [≡] [← 首页]    Flutter > 底层原理.md    [A+] [🌙] │
└────────────────────────────────────────────────────┘
```

- **桌面端收起行为**：点击工具条的 `≡` 按钮切换侧边栏显隐。收起时 `.note-sidebar-wrap` 的 `width` 过渡到 `0`（而非 48px），图标从 `◀` 变为 `≡`。
  - 备选：保留 48px 窄条。
  - 取舍：窄条没有实际功能（按钮已在工具条），只是浪费空间。

- **移动端**：工具条固定顶部，`≡` 按钮触发抽屉。取消独立的 `note-sidebar-toggle-mobile` 浮动按钮。预览区不再需要 `padding-top: 4rem`。
  - 备选：保留浮动按钮但放在工具条同一行。
  - 取舍：既然有工具条，浮动按钮完全冗余。

- **面包屑截断**：面包屑容器用 `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`，窄屏时自动省略中间路径。

## Risks / Trade-offs

- **[风险]** 顶部工具条固定后占用垂直空间。
  **缓解：** 工具条高度约 44px，与移动端原来的浮动按钮 + padding-top: 4rem (64px) 相比反而节省了 20px。

- **[风险]** 面包屑在移动端被截断可能影响可读性。
  **缓解：** 移动端只显示最后两级路径（如 `… > 底层原理.md`），保证关键信息可见。
