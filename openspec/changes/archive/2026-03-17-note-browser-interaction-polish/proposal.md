## Why

笔记页的交互存在多处别扭的体验：

1. **桌面端收起态割裂**：侧边栏收起后只剩 48px 窄条 + 一个 `◀` 按钮，主题切换、大字模式、返回首页全部消失，用户失去了对页面的控制。
2. **移动端浮动按钮与 topbar 冲突**：`≡` 目录按钮固定在左上角，预览区要 `padding-top: 4rem` 给它让路，导致首屏浪费空间；和 `← 返回首页` 不在同一视觉层级。
3. **返回首页位置不统一**：其他页面都在顶部 `.glass-nav` 中，笔记页却放在预览区内容区的 topbar 里，风格不统一且占用内容空间。
4. **topbar 冗余**：返回首页 + 面包屑挤在一行，移动端宽度不够时换行难看。
5. **功能按钮随侧边栏消失**：主题切换和大字模式按钮藏在侧边栏 header 中，收起侧边栏后无法访问。

## What Changes

- **新增顶部固定工具条**：在页面顶部添加 `.glass-nav` 工具条，包含返回首页、目录展开/收起切换、面包屑、主题切换和大字模式按钮。这条工具条始终可见。
- **移除预览区内的 topbar**：返回首页和面包屑移入顶部工具条后，预览区内不再需要 topbar。
- **重做桌面端收起态**：收起侧边栏后，顶部工具条的目录按钮变为展开触发器（取代窄条里的 `◀`），侧边栏完全隐藏（width: 0）而非留 48px 窄条。
- **统一移动端目录按钮**：将浮动的 `≡` 按钮移入顶部工具条，取消 `position: fixed` 的独立浮动按钮和 `padding-top: 4rem` 的让路空间。
- **面包屑移入工具条**：面包屑显示在工具条中部（桌面端）或省略（移动端宽度不足时自动截断）。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `note-browser`：导航结构、侧边栏收起交互、移动端目录触发方式、面包屑位置。

## Impact

- 核心受影响文件：`note/index.html`（HTML 结构调整）、`assets/css/note-browser.css`（大量布局修改）、`assets/js/note-browser.js`（事件绑定调整）。
- 不改变笔记加载、搜索、highlight.js 等功能逻辑。
