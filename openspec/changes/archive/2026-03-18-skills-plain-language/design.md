## Context

当前 skills.html 每个 Step 的结构是：标题 → 简短说明 → 代码块 → 折叠知识点。
非程序员在遇到第一个代码块时就会放弃阅读。

## Goals / Non-Goals

**Goals:**
- 非程序员读完主线文字能理解 Skill 是什么、怎么用、有什么价值
- 程序员展开折叠区后能看到所有代码和技术细节
- 页面结构（5 步、sidebar、底部导航）完全不变

**Non-Goals:**
- 不改变 CSS/JS
- 不删除任何现有代码示例（只是移到折叠区）
- 不改变其他页面

## Decisions

### 每个 Step 的新结构

```
Step N: 标题
  ├── 生活类比 / 通俗解释（2-3 段纯文字，0 行代码）
  ├── 可视化图表或类比图（如果适用）
  ├── ▶ 技术细节（折叠）← 代码示例移到这里
  └── ▶ 知识点详解（折叠）← 保留原有内容
```

### 各步类比设计

| Step | 当前开头 | 改为 |
|------|---------|------|
| 1 Skill 是什么 | "Tool 是函数，Skill 是更高层抽象" | "Tool 像一个工具（扳手），Skill 像一本操作手册（汽车保养手册）" |
| 2 编写 SKILL.md | "YAML 头部 + Markdown 正文" | "像写一份说明书：封面信息（名称、用途）+ 操作步骤" |
| 3 加载使用 | 代码开头 | "像把说明书放进工具箱——Agent 自动翻阅、按需使用" |
| 4 实战示例 | 直接代码 | "用一个真实例子：教 Agent 像资深程序员一样审查代码" |
| 5 脚本命令 | 目录结构 + 代码 | "说明书里可以附带小工具——自动完成重复操作的脚本" |

## Risks / Trade-offs

- **[内容变长]** → 折叠机制保证页面不会显得过长，主线反而更简洁
- **[已有代码移位]** → 代码全部保留，只是位置从主线移到折叠区
