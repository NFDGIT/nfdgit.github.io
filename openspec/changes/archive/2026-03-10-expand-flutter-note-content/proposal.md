# 扩充 Flutter 笔记内容至约 5 倍

## Why

当前 `note/Flutter/` 下三篇文档（底层原理、面试、高阶用法）合计约 160 行，内容偏精简短小，读者「不够看」、难以作为系统复习或查漏补缺的素材。将整体内容扩充至**至少约 5 倍**（以当前正文行数或等效字数为基线），在保持生动简练文风的前提下增加深度与覆盖面，提升笔记的实用价值。

## What Changes

- **扩充现有三篇文档**：
  - **底层原理.md**：在现有框架上增加小节与细节（如 Layer、Compositing、Key 与更新、InheritedWidget、BuildOwner、PipelineOwner、Flutter 与 Skia/Impeller 等），辅以示意与小结，目标约为当前的 5 倍篇幅。
  - **面试.md**：大幅增加题目数量与分类（原理、Dart 语言、状态管理、网络与存储、性能、工程与调试、综合场景等），每题保留要点或参考答案，目标约为当前的 5 倍篇幅。
  - **高阶用法.md**：扩展状态管理各方案对比与示例、性能优化清单与案例、Platform Channel 详解、混合开发步骤、插件开发要点、测试与 CI 等，目标约为当前的 5 倍篇幅。
- **可选**：若单文件过长影响阅读，可拆分为多文件（如 面试-原理.md、面试-实践.md）或增加「最佳实践」「性能优化」「资源与链接」等独立文档，并在 README 中更新导航。
- 内容继续从网络（官方文档、优质博客、面试汇总）对比择优，保持语言生动、言简意赅。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `flutter-note-doc`：在现有「分类存在且文风生动简练」基础上，增加「内容体量至少约为扩充前 5 倍」的约定（见本 change 的 delta spec）。

## Impact

- **note/Flutter/**：仅修改或扩充现有 Markdown 及可选新增文档；README 导航可能更新。
- 不改变站点代码、构建或笔记浏览器逻辑。
