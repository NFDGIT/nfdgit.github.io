## Why

笔记浏览器功能完善，但排版和交互体验明显落后于站点其他已优化页面：侧边栏视觉层级不够清晰（目录/文件缺乏图标区分、展开/折叠缺少视觉强调）；预览区 Markdown 排版单调（段落间距紧凑、标题缺少锚点、代码块无语法高亮、引用块和表格样式粗糙）；缺少面包屑导航和搜索能力；移动端抽屉虽能用但缺少手势支持和过渡细节。

## What Changes

- 侧边栏视觉增强：为目录节点和文件节点添加类型图标（文件夹/Markdown/HTML）；活跃节点增加左侧色条指示；收起态显示可识别的图标列。
- 预览区 Markdown 排版重做：优化标题层级样式（字号/间距/底线分割）、段落行高与间距、引用块样式（左侧色条 + 背景）、表格样式（斑马行 + 边框）、图片最大宽度约束与圆角。
- 引入代码语法高亮：通过 CDN 加载 highlight.js + 自动检测语言，渲染 Markdown 中的代码块。
- 预览区添加面包屑路径显示（当前打开文件的目录 > 文件名）。
- 预览区空态（placeholder）改为更友好的插画/图标 + 提示文案。
- 侧边栏底部添加文件统计（x 个目录、y 个文件）。
- 移动端侧边栏抽屉优化：背景模糊增强、滑入弹性曲线、关闭按钮更明显。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `note-browser`：排版样式、侧边栏视觉、面包屑导航、代码高亮、空态设计、移动端抽屉优化。

## Impact

- 核心受影响文件：`assets/css/note-browser.css`（大量样式新增/修改）、`note/index.html`（引入 highlight.js CDN、面包屑 HTML）、`assets/js/note-browser.js`（面包屑更新、高亮调用、文件统计渲染）。
- 新增 CDN 依赖：highlight.js（轻量，仅 CSS + 自动检测 JS）。
- 不改变 manifest 结构和文件加载逻辑。
