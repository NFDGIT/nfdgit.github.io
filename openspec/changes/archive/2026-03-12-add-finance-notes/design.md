## Context

站点笔记已有 AI、Flutter、IT、iOS 等专题目录，均为 `note/<主题>/` 下 md + README 导航，由 `generate-note-manifest.js` 扫描生成侧栏。金融专题为全新目录，无既有结构可迁，需从零建立目录与首期文档。

## Goals / Non-Goals

**Goals:**

- 建立 `note/金融/` 目录及 README 索引；至少包含「金融知识入门」「理财入门」「与金融人士交流」三类入口（可为单篇或多篇），内容通俗、面向非专业读者。
- 文风统一为通俗易懂：少行话、多比喻与短句，关键概念配「一句话说清」；不代替专业建议，可注明「仅供参考、具体以持牌机构为准」。

**Non-Goals:**

- 不提供实时行情、不接金融 API、不做投资建议或合规背书；不替代线下与持牌机构沟通。内容为静态科普与入门整理。

## Decisions

1. **目录名**  
   使用 `金融` 而非 `Finance`，与站点中文为主、以及「保险」等已有入口命名一致，便于在笔记浏览器中一眼识别。

2. **首期文档结构**  
   至少三块：**金融知识入门**（概念、术语、常见产品类型）、**理财入门**（储蓄、保险、基金/股票等基础与注意点）、**与金融人士交流**（听懂行话、如何提问、常见场景与避坑）。每块可单文件（如 `金融知识入门.md`）或拆成多文件（如 `理财-储蓄与保险.md`、`理财-投资基础.md`），以 README 表格链接即可；首期优先单文件保证「有且可读」，后续再按需拆分。

3. **通俗化原则**  
   术语首次出现时用括号或短句解释；避免长段公式与法规原文堆砌；每节可有「一句话」小结；可选「和银行/客户经理沟通时他们会怎么说」类小框，帮助读者对上线下沟通。

4. **Manifest**  
   与现有笔记一致：新增目录与 md 后执行 `node scripts/generate-note-manifest.js`，无需改脚本逻辑（已按目录扫描）。

## Risks / Trade-offs

- **Risk**：金融内容涉及合规与责任，若表述不当易被误解为投资建议。  
  **Mitigation**：文中明确「入门科普、不构成建议」「具体以持牌机构与产品条款为准」；不推荐具体产品与收益率承诺；涉及风险处写明「可能亏损」「需根据自身情况」等。
- **Trade-off**：首期不追求覆盖「所有」金融知识，先建立结构+核心概念+理财与沟通入口，后续按需补充防骗、资产配置、税务等。
