# billiards-game（变更增量）

## ADDED Requirements

### Requirement: 台球游戏支持 3D 显示模式

系统 SHALL 支持以 3D 方式渲染台球桌、球与球袋：使用 WebGL 在三维空间中绘制球桌桌面、边框、六个球袋以及母球与彩球，相机可为俯视或适度倾斜，光照与材质使球与桌面具有立体感。3D 模式下玩法、物理规则（碰撞、摩擦、进袋）、积分与通关判定 SHALL 与 2D 模式一致。

#### Scenario: 用户选择 3D 模式
- **WHEN** 用户在台球游戏页面选择或切换到 3D 模式
- **THEN** 系统 SHALL 以 3D 场景渲染台球桌、球与球袋，且击球、瞄准、进袋、积分与重置等行为与 2D 模式一致

#### Scenario: 3D 模式下球桌与球可见
- **WHEN** 游戏处于 3D 模式且页面已加载
- **THEN** 用户 SHALL 能看到三维的台球桌（桌面、边框、球袋）以及母球与未进袋的彩球，球体与桌面具有立体视觉效果

#### Scenario: 3D 模式下可瞄准与击球
- **WHEN** 用户在 3D 模式下在球桌区域拖拽瞄准并释放
- **THEN** 系统 SHALL 按与 2D 相同的规则更新母球速度与方向，并显示瞄准线（或等效反馈）；碰撞、进袋与积分逻辑不变

### Requirement: 用户可在 2D 与 3D 模式之间切换

系统 SHALL 在台球游戏页面提供 2D/3D 视角切换入口（如按钮或选项），用户切换后 SHALL 在对应模式下看到球桌与球的渲染；切换时游戏状态（球位、积分、是否通关）SHALL 保持不重置。

#### Scenario: 从 2D 切换到 3D
- **WHEN** 用户当前处于 2D 模式并触发切换至 3D
- **THEN** 系统 SHALL 切换到 3D 渲染，且当前局内球的位置、积分与游戏状态保持不变

#### Scenario: 从 3D 切换到 2D
- **WHEN** 用户当前处于 3D 模式并触发切换至 2D
- **THEN** 系统 SHALL 切换到 2D 渲染，且当前局内球的位置、积分与游戏状态保持不变

#### Scenario: 3D 不可用时的降级
- **WHEN** WebGL 不可用或 3D 库加载失败
- **THEN** 系统 SHALL 保持或回退到 2D 模式，并可选择向用户提示 3D 不可用
