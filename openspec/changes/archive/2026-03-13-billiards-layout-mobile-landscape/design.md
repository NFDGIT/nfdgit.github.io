## Context

当前台球页采用“信息面板 + 玩法说明 + 球桌”并列布局，在中小屏设备上球桌可用面积被明显压缩，视觉重心分散。用户希望球桌成为主区域，次要功能弱化，并在移动端支持横屏优先展示。

## Goals / Non-Goals

**Goals:**

- 提升球桌在首屏中的占比与视觉权重，确保主要操作围绕球桌进行。
- 将非常用操作收纳到“选项”入口，保留核心操作（开始/重置/2D/3D）直达。
- 在移动端横屏场景下提供更高利用率的布局，减少无关信息遮挡球桌。

**Non-Goals:**

- 不调整台球物理规则、计分逻辑与胜负判定。
- 不引入新框架或构建流程。
- 不重做整站导航样式，仅针对台球页局部优化。

## Decisions

- **球桌优先布局**：将球桌容器提升为主网格区域，信息与说明改为次要块（紧凑化/折叠化），避免与球桌争夺首屏空间。
- **功能分层**：把非常用功能（如玩法细节、辅助说明、扩展选项）放入“选项”面板（details/弹层/折叠区任一轻量实现），默认收起；核心按钮保持可见。
- **横屏优先策略**：为移动端 `landscape` 增加独立样式分支，优先放大球桌宽高比并压缩侧栏；必要时将说明区移入选项面板。
- **渐进增强**：在不破坏现有 DOM 语义的前提下改造结构和样式，JS 仅负责选项面板开合等轻交互（如需要），以降低回归风险。

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 过度弱化信息区导致新手难以上手 | 首次访问可默认展开简版说明，之后保持用户手动收起状态。 |
| 横屏优化影响竖屏排版 | 使用媒体查询分层控制，仅在 `orientation: landscape` 且窄高场景启用横屏优先规则。 |
| 结构调整影响既有按钮脚本绑定 | 优先保留既有元素 ID/class，不破坏事件选择器；变更后做按钮回归验证。 |
