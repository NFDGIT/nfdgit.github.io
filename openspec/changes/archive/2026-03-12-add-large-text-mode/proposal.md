## Why

部分用户（如视力较弱或小屏阅读）希望站点正文与标题能整体放大，提升可读性。当前站点无统一「大字模式」开关，增加该能力可在不改变布局逻辑的前提下，通过一次切换放大全站文字，兼顾可访问性与使用习惯。

## What Changes

- 新增「大字模式」开关：在支持主题切换的页面上提供可见控件（如与主题按钮并列的图标按钮），用户点击后在全站或当前模块内启用/关闭大字模式。
- 大字模式生效时：通过 CSS 变量或 data 属性放大正文、标题等字体尺寸（例如约 1.2–1.25 倍基准），保持行高与可读性。
- 偏好持久化：用户选择的大字模式开/关状态写入 localStorage，下次访问同设备时自动恢复。
- 可选：首次访问时尊重系统字体缩放（如 `prefers-contrast` / 大字号偏好），若难以可靠检测则可仅做「用户显式开关 + 持久化」。

## Capabilities

### New Capabilities

- `large-text-mode`：定义站点「大字模式」的交互与表现——开关控件、开/关状态下的字体放大策略、localStorage 持久化及（若有）与系统偏好的一致性。

### Modified Capabilities

- 无（theme 仅负责亮/暗色，不修改其需求；大字模式为独立能力）。

## Impact

- **assets/js**：新增或扩展脚本（如 `theme.js` 旁新增 `large-text.js`，或在同一脚本内增加大字模式读写与按钮绑定），与现有主题脚本协同、避免冲突。
- **assets/css**：共享样式（如 `home.css`、`note-browser.css` 等）中增加基于 data 属性或类的字体缩放规则，保证首页、笔记、游戏等主要模块在开启大字模式时一致放大。
- **页面**：带主题切换的页面（如首页、笔记页、游戏目录等）需在 header 或工具栏区域增加大字模式开关入口，并引入对应脚本与样式。
