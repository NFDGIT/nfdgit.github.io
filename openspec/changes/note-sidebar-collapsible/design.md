## 背景

笔记页使用 `note-browser.css` 和 `note-browser.js`。移动端（≤768px）目录为抽屉，通过 `note-sidebar-toggle` 和 `note-sidebar-open` 控制；桌面端目录固定 260px 宽，无收起能力。

## 目标 / 非目标

**目标：**
- 桌面端目录可收起，收起后仅显示展开按钮或窄条
- 移动端保持现有抽屉行为
- 切换控件在两端均可见且易用

**非目标：**
- 不改变移动端现有交互
- 不引入新依赖

## 决策

1. **桌面端收起态**：目录宽度设为 0 或极小（如 48px 仅放图标），通过 CSS class（如 `note-sidebar-collapsed`）控制
2. **切换控件**：复用 `note-sidebar-toggle`，桌面端始终显示在侧边栏边缘或预览区左侧；移动端保持左上角固定
3. **展开按钮**：桌面端收起后，在左侧显示「≡」或类似图标，点击展开
4. **持久化**：可选，localStorage 存 `note-sidebar-collapsed`，页面加载时恢复
5. **响应式**：桌面端与移动端共用同一套 JS 逻辑，通过 `isMobile()` 分支处理不同行为

## 风险与权衡

- **[风险]** 桌面端收起后，新访客可能不知道如何展开
  - **缓解**：展开按钮明显可见，或首次访问默认展开
