## Context

保险页当前是一个极简的图片落地页，结构为：导航浮窗 + 两张全宽产品图 + 左侧固定联系浮窗。全部通过内联样式实现，无独立 CSS，不适配暗色主题，移动端体验差。站点其他页面（首页、游戏目录）已经有了成熟的卡片 + 动画 + 主题变量体系，保险页需要对齐。

## Goals / Non-Goals

**Goals:**

- 用现代卡片式布局重建保险页，复用 `home.css` 的变量、卡片、动画工具类。
- 添加 hero 区（标题 + 副标题 + 联系按钮）替代无结构的图片堆叠。
- 产品图片包装为带说明文字的卡片，支持悬停效果。
- 联系方式整合为 hero 区按钮 + 底部浮动条（移动端），替代左侧白色浮窗。
- 完全消除内联样式，新建 `insurance/insurance.css`。
- 暗色主题完全适配。

**Non-Goals:**

- 不更换产品图片素材。
- 不添加表单或支付功能。
- 不改变 URL 参数读取逻辑。

## Decisions

- **页面结构**：`hero 区` → `产品卡片区` → `联系区`。hero 区用渐变背景 + 大标题 + 副标题 + CTA 按钮组。产品卡片用 `.home-card` 基础样式扩展。
  - 备选：保留全宽图片 + 底部文字。
  - 取舍：卡片更适合主题适配和响应式，且与站点风格一致。

- **联系方式**：hero 区放两个按钮（电话 + QQ），移动端底部追加固定条。不再用左侧浮窗。
  - 备选：保留浮窗但改样式。
  - 取舍：浮窗在窄屏遮挡内容，底部条更符合移动端惯例。

- **CSS 组织**：新建 `insurance/insurance.css`，只写保险页特有样式。通用变量、卡片、动画来自 `home.css`。
  - 备选：全写在 `home.css`。
  - 取舍：保险页样式不被其他页面需要，单独维护更清晰。

- **入场动画**：hero 用 `.anim-fade-in`，产品卡片用 `.anim-slide-up` + `stagger-*`。

## Risks / Trade-offs

- **[风险]** 重写后原有的电话/QQ 点击入口被遗漏。
  **缓解：** 逐一保留 `tel:` 和 `http://wpa.qq.com` 链接，验证可点击。

- **[风险]** 产品图片比例在卡片内变形。
  **缓解：** 使用 `object-fit: cover` + 固定宽高比约束。
