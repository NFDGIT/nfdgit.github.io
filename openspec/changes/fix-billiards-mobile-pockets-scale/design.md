## Context

当前台球页在移动端存在两类核心问题：其一，六个球袋在缩放后与桌边几何关系偏移，视觉与碰撞预期不一致；其二，球体与台面的比例失衡，导致触控操作和进袋判断体验变差。问题来源于移动端逻辑尺寸、缩放因子、球袋坐标与 2D/3D 渲染映射未统一约束。

## Goals / Non-Goals

**Goals:**

- 修正移动端六个球袋布局，使其在竖屏/横屏都与桌边几何关系正确。
- 统一移动端球体半径与台面尺寸比例，保证观感与可操作性。
- 保证 2D 与 3D 在移动端切换时，球袋位置与球台比例语义一致。

**Non-Goals:**

- 不改变物理碰撞、摩擦、进袋计分等核心规则。
- 不引入新渲染依赖或构建工具。
- 不重做整页信息架构，仅修正移动端布局与比例逻辑。

## Decisions

- **统一移动端几何参数源**：将球袋位置、球半径、桌边宽度的计算集中到同一套逻辑尺寸函数，2D 与 3D 都消费该结果，避免分叉。  
  - 备选方案：分别调 2D 与 3D 的常量。  
  - 取舍：短期快但易再次漂移，不采用。

- **按屏幕区间重设比例，而非单点放大**：针对窄屏/横屏使用受控比例区间（含上限下限），避免球过大遮挡袋口或过小难操作。  
  - 备选方案：固定球半径。  
  - 取舍：不同设备差异大，固定值不可兼容。

- **袋口与边界联动校验**：在更新尺寸时同步更新袋口吸附阈值与渲染半径，确保“看到的袋口位置”与“可进袋区域”一致。  
  - 备选方案：仅修视觉不修阈值。  
  - 取舍：会造成视觉命中与逻辑命中不一致，不采用。

- **移动端回归优先**：把竖屏与横屏列为必测场景，覆盖 2D/3D 切换后的比例一致性。

## Risks / Trade-offs

- **[风险]** 调整比例后影响既有手感。  
  **缓解：** 通过上下限控制与多设备回归，维持“更准但不突变”的体验。

- **[风险]** 2D/3D 共用参数导致一处调整影响两端。  
  **缓解：** 将参数集中并写清注释，回归时同时验证两种模式。

- **[风险]** 横屏设备可用高度不足导致压缩失真。  
  **缓解：** 保留最小可用尺寸阈值，必要时优先保证桌面比例与操作区可达。

## Migration Plan

- 无数据迁移。
- 发布前执行移动端回归：竖屏/横屏下六袋位置、球台比例、进袋判定、2D/3D切换一致性。
- 如出现比例回归，可回退到本次变更前的布局参数与缩放策略。
