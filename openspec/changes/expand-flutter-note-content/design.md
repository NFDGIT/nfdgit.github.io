# Flutter 笔记扩充至约 5 倍 — 设计

## Context

当前 Flutter 三篇正文合计约 161 行（底层原理 46、面试 63、高阶用法 52）。用户要求「至少 5 倍」，即目标合计约 **800+ 行**（或等效字数），在保持生动简练的前提下增加深度与题量。

## Goals / Non-Goals

**Goals:**

- 将底层原理、面试、高阶用法三篇各自扩充到约 5 倍于当前篇幅（可按行数或字数估算，允许略超或略欠以照顾结构）。
- 扩充方式以「加小节、加题目、加要点与示例」为主，不注水；内容仍从网络对比择优。
- 保持小标题、列表、表格与「一句话」小结，文风不变。

**Non-Goals:**

- 不降低「言简意赅」标准；不改为长文堆砌。
- 不强制新增独立文档；若实现时单文件过长（如面试 >300 行），可拆分为多文件并在 README 中链入。

## Decisions

1. **体量基线**  
   以当前三篇正文行数之和为基线（约 161 行），5 倍即约 **805 行**。分配建议：底层原理约 230 行、面试约 315 行、高阶用法约 260 行（可微调），合计达标即可。

2. **底层原理扩充方向**  
   增加：Layer 与合成、Key 与 Element 复用、InheritedWidget 与依赖注入、BuildOwner/PipelineOwner 角色、Flutter 与 Skia/Impeller、约束传递与布局边界、RepaintBoundary 原理、帧调度与 VSync 等；每块保持「一句话 + 要点」风格。

3. **面试扩充方向**  
   增加题目数量与分类：Dart（isolate、async、空安全、mixin）、Widget 生命周期、Key、状态管理对比与场景、网络（Dio/http）、本地存储、路由、性能（jank、Overdraw）、调试与 DevTools、工程（pub、多环境）等；每题给出要点或短答，便于复习。

4. **高阶用法扩充方向**  
   状态管理：各方案代码示例与选型表、Riverpod/Bloc 核心概念；性能：清单 + 常见坑 + RepaintBoundary/const 详解；Platform Channel：MethodChannel/EventChannel 示例与注意点；混合开发：嵌入流程与通信；插件：创建与发布要点；测试：单元/Widget/集成简述；CI：常见流水线要点。

5. **可选拆分**  
   若面试.md 超过约 300 行，可拆为 面试-原理.md、面试-实践.md 等，README 中增加链接；其他文档以单文件扩充为主。

## Risks / Trade-offs

- **[取舍]** 篇幅大增后单文件滚动长 → 用清晰小标题与目录感的小结缓解；必要时按上一条拆分。
- **[取舍]** 网络内容时效性 → 继续注明「以官方与最新资料为准」，关键结论附来源或链接。
