# billiards-game（变更增量）

## MODIFIED Requirements

### Requirement: 台球游戏支持 3D 显示模式

系统 SHALL 支持以 3D 方式渲染台球桌、球与球袋：使用 WebGL 在三维空间中绘制球桌桌面、边框、六个球袋以及母球与彩球，相机可为俯视或适度倾斜，光照与材质使球与桌面具有立体感。3D 模式下玩法、物理规则（碰撞、摩擦、进袋）、积分与通关判定 SHALL 与 2D 模式一致。系统 SHALL 保证 3D 视图中的可见边界与逻辑桌面边界一致，避免桌边被裁切或球在边界附近不可见。

#### Scenario: 用户选择 3D 模式
- **WHEN** 用户在台球游戏页面选择或切换到 3D 模式
- **THEN** 系统 SHALL 以 3D 场景渲染台球桌、球与球袋，且击球、瞄准、进袋、积分与重置等行为与 2D 模式一致

#### Scenario: 3D 模式下球桌与球可见
- **WHEN** 游戏处于 3D 模式且页面已加载
- **THEN** 用户 SHALL 能看到三维的台球桌（桌面、边框、球袋）以及母球与未进袋的彩球，球体与桌面具有立体视觉效果，且球桌边缘不被视口裁切

#### Scenario: 3D 模式下可瞄准与击球
- **WHEN** 用户在 3D 模式下在球桌区域拖拽瞄准并释放
- **THEN** 系统 SHALL 按与 2D 相同的规则更新母球速度与方向，并显示瞄准线（或等效反馈）；碰撞、进袋与积分逻辑不变

#### Scenario: 3D 模式窗口尺寸变化后仍完整可见
- **WHEN** 用户在 3D 模式下发生窗口缩放、设备方向变化或容器尺寸变化
- **THEN** 系统 SHALL 重新同步渲染尺寸与相机参数，保证球桌边框和袋口仍完整可见，不出现边缘缺失

#### Scenario: 球在边界附近运动时保持可见
- **WHEN** 球在 3D 模式下运动到逻辑桌面边界附近并发生反弹
- **THEN** 球 SHALL 在可见区域内持续可见，直到离开该区域或进袋，不得出现“仍在桌面逻辑范围但画面消失”
