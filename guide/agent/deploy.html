Agent 可以怎么部署？

概览选型

你的 Agent 目前只能在自己电脑上通过命令行运行。要让别人也能用，你有三个选择：

🔌
API 服务
给开发者调用
/
💬
聊天机器人
接入微信/钉钉/飞书
/
🌐
Web 应用
做一个网页让用户聊

API 服务适合其他程序调用（比如你的 App 后端）。聊天机器人适合让普通用户在微信/钉钉里直接使用。Web 应用适合做一个独立的产品页面。

最基础也最通用的方式是先做成 API——有了 API，就能接到任何地方。

2

用 FastAPI 包装成 API

FastAPIHTTP

FastAPI 是 Python 最流行的 Web 框架之一。用它把你的 Agent 包装成一个 HTTP 接口，别人就能通过网络调用。

原来：你在命令行输入问题 → Agent 在终端回答。
现在：任何程序发一个 HTTP 请求 → Agent 通过网络回答。

bash
# 安装依赖
pip install fastapi uvicorn
python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class ChatRequest(BaseModel):
    message: str

class ChatResponse(BaseModel):
    reply: str

@app.post("/chat")
async def chat(req: ChatRequest):
    reply = run_agent(req.message)  # 你之前写的 Agent 函数
    return ChatResponse(reply=reply)

# 启动：uvicorn main:app --host 0.0.0.0 --port 8000

调用示例

bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "北京天气怎么样？"}'

接入聊天平台

企业微信钉钉飞书

有了 API 之后，接入聊天平台就是"收到消息 → 调 Agent → 回复消息"的流程。

各平台的差别在于怎么收消息和怎么发消息——它们都提供了 Webhook（一个回调网址），用户发消息时平台会把消息发到你的网址。

最快落地方式：如果不想自己对接平台 API，可以用 OpenClaw（教程前面学过）——它已经帮你封装好了 WhatsApp、Telegram、Discord、钉钉等平台的接入。
python
import requests

WEBHOOK_URL = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY"

def send_to_wechat(message):
    """通过 Webhook 发送消息到企业微信群"""
    requests.post(WEBHOOK_URL, json={
        "msgtype": "text",
        "text": {"content": message}
    })

# 收到用户消息后
user_msg = "帮我查一下今天的会议安排"
reply = run_agent(user_msg)
send_to_wechat(reply)

部署清单

  • 服务器：云服务器（阿里云/腾讯云）或 Railway/Render 等平台
  • 域名（可选）：让 API 地址更好看
  • HTTPS：平台 Webhook 通常要求 HTTPS
  • 环境变量：API Key 不要写在代码里，用环境变量
  • 日志：记录每次对话，方便排查问题
