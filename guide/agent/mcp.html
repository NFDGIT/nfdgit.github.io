MCP 是什么？

MCP 协议 标准化

MCP（Model Context Protocol）是一套开放协议，定义了 AI 应用与外部工具/数据源之间的通信标准。

🤖
Agent
MCP Client
⇄
🔌
MCP Server
暴露工具和资源
⇄
🗄️
外部服务
数据库、API 等

类比：USB 是硬件的标准接口，MCP 是 AI 工具的标准接口。一个 MCP Server 可以被任何支持 MCP 的 Agent 使用。

MCP 的核心概念

  • MCP Server：一个进程，暴露 tools（工具）和 resources（资源）供 Agent 调用。
  • MCP Client：Agent 端的连接器，负责发现和调用 Server 的工具。
  • Transport：通信方式，常用 stdio（子进程）或 SSE（HTTP）。
  • Tools：Server 提供的可调用函数，类似 Function Calling 的工具定义。
  • Resources：Server 提供的只读数据（文件、数据库记录等）。

为什么用 MCP？

  • 不用为每个 Agent 重复写工具——一个 MCP Server 所有 Agent 共享。
  • 社区已有大量现成 Server：GitHub、Slack、PostgreSQL、文件系统等。

安装一个 MCP Server

安装 filesystem

以官方的 filesystem MCP Server 为例——它让 Agent 能读写本地文件。

bash 
# 安装 MCP Python SDK
pip install mcp

# 用 npx 启动 filesystem server（需要 Node.js）
npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/dir

启动后 Server 通过 stdio 通信，等待 Client 连接。

其他常用 MCP Server

  • @modelcontextprotocol/server-github — 操作 GitHub 仓库、issues、PR
  • @modelcontextprotocol/server-postgres — 查询 PostgreSQL 数据库
  • @modelcontextprotocol/server-slack — 发送/读取 Slack 消息
  • 完整列表见 MCP Servers 仓库

在 Agent 中连接 MCP

MCP Client Python SDK

用 MCP Python SDK 连接 Server，获取工具列表后转换为 OpenAI 格式传给模型。

python 
import asyncio
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def get_mcp_tools():
    """连接 MCP Server 并获取工具列表"""
    server_params = StdioServerParameters(
        command="npx",
        args=["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    )

    async with stdio_client(server_params) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()

            # 列出所有可用工具
            tools_result = await session.list_tools()

            # 转换为 OpenAI tools 格式
            openai_tools = []
            for tool in tools_result.tools:
                openai_tools.append({
                    "type": "function",
                    "function": {
                        "name": tool.name,
                        "description": tool.description,
                        "parameters": tool.inputSchema
                    }
                })

            return openai_tools

# 获取工具后传给 Agent
tools = asyncio.run(get_mcp_tools())
print(f"从 MCP Server 获取到 {len(tools)} 个工具")
核心思路：MCP 自动提供工具的 schema，你不用手写 JSON Schema——Server 已经定义好了。只需把 MCP 工具转成 OpenAI 格式，Agent 循环不用改。

调用 MCP 工具

call_tool 完整流程

当模型决定调用工具时，通过 MCP Session 执行：

python 
async def call_mcp_tool(session, name, arguments):
    """通过 MCP Session 调用工具"""
    result = await session.call_tool(name, arguments=arguments)
    return result.content[0].text

# 在 Agent 循环中替换原来的本地函数调用：
# 原来：result = tool_functions[name](**args)
# 现在：result = await call_mcp_tool(session, name, args)

MCP vs 手写工具

  • 手写：你自己定义 schema + 实现函数。灵活但需要维护。
  • MCP：Server 提供 schema + 实现。即插即用，社区共享。
  • 可以混用：一些工具用 MCP，一些自己实现，合并到同一个 tools 数组。
