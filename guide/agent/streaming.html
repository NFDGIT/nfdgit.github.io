流式输出：实时打字效果

让 Agent 像人一样逐字"打"出回答，而不是等半天一次性蹦出来。

  1. 为什么要流式
  2. 实现流式
  3. 流式+工具调用
1

为什么需要流式输出？

体验类比

想象你问朋友一个问题。你更喜欢哪种回答方式？

⏳
等 10 秒
然后一次性出现整段话
vs
⌨️
立刻开始
一个字一个字"打"出来

大多数人更喜欢第二种——虽然总时间一样，但你马上就能看到回答在生成，不会觉得 Agent 卡死了。ChatGPT、Claude 聊天界面就是这种效果。

技术上，这叫流式输出（Streaming）——服务器每生成一个词就立刻发给你，不用等全部生成完。

2

让你的 Agent 支持流式

stream=True实现

改动非常小：只需要加一个参数 stream=True，然后逐块读取回复。

原来是"等 Agent 说完 → 一次性拿到全部文字"，现在是"Agent 边说边传 → 你边收边显示"。

非流式（原来的方式）

python
# 等全部生成完才返回
response = client.chat.completions.create(
    model="gpt-4o-mini", messages=messages
)
print(response.choices[0].message.content)

流式（逐字输出）

python
# 加上 stream=True，逐块接收
stream = client.chat.completions.create(
    model="gpt-4o-mini", messages=messages,
    stream=True
)

for chunk in stream:
    content = chunk.choices[0].delta.content
    if content:
        print(content, end="", flush=True)

print()  # 最后换行

关键区别

  • stream=True：返回一个迭代器，逐块产出
  • 每个 chunk 的内容在 delta.content（不是 message.content）
  • flush=True 确保每个字立刻显示，不被缓冲
3

流式中的工具调用

tool_calls流式拼接

当 Agent 决定调用工具时，工具调用信息也是分块传来的——你需要把碎片拼起来，拼完后执行工具。

就像收到一封被拆成很多片的信：先攒齐所有碎片，拼出完整内容，再按信里的指示行动。

什么时候用流式？ 面向用户的交互界面（聊天、Web 应用）建议用流式。后台批处理、自动化脚本用非流式更简单。
python
import json

def run_stream_agent(user_input):
    messages = [
        {"role": "system", "content": "你是智能助手。"},
        {"role": "user", "content": user_input}
    ]

    for _ in range(10):
        stream = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages,
            tools=tools, stream=True
        )

        content_parts = []
        tool_calls_data = {}

        for chunk in stream:
            delta = chunk.choices[0].delta

            # 文本内容 → 立刻打印
            if delta.content:
                print(delta.content, end="", flush=True)
                content_parts.append(delta.content)

            # 工具调用 → 攒碎片
            if delta.tool_calls:
                for tc in delta.tool_calls:
                    idx = tc.index
                    if idx not in tool_calls_data:
                        tool_calls_data[idx] = {
                            "id": "", "name": "", "arguments": ""
                        }
                    if tc.id:
                        tool_calls_data[idx]["id"] = tc.id
                    if tc.function and tc.function.name:
                        tool_calls_data[idx]["name"] = tc.function.name
                    if tc.function and tc.function.arguments:
                        tool_calls_data[idx]["arguments"] += tc.function.arguments

        # 没有工具调用 → 输出完毕
        if not tool_calls_data:
            print()
            return "".join(content_parts)

        # 执行工具调用
        assistant_msg = {"role": "assistant", "content": "".join(content_parts) or None,
                         "tool_calls": []}
        for idx in sorted(tool_calls_data):
            tc = tool_calls_data[idx]
            assistant_msg["tool_calls"].append({
                "id": tc["id"], "type": "function",
                "function": {"name": tc["name"], "arguments": tc["arguments"]}
            })

        messages.append(assistant_msg)
        for tc in tool_calls_data.values():
            fn = tool_functions.get(tc["name"])
            result = fn(**json.loads(tc["arguments"])) if fn else '{"error":"未知工具"}'
            messages.append({"role": "tool", "tool_call_id": tc["id"], "content": result})
