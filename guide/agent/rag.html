RAG 是什么？

概念开卷考试

AI 模型的知识来自训练数据——但它不知道你公司的产品文档、你的个人笔记、最新的新闻。直接问它，它可能会编造答案。

RAG（检索增强生成）的思路很简单：让 Agent 先翻参考资料，再回答问题。

❓
用户提问
"退货流程是什么？"
→
🔍
检索资料
从文档中找到相关段落
→
🤖
生成回答
基于资料回答，不编造

就像开卷考试：学生可以翻书找答案，而不是全靠记忆。翻了书的答案通常更准确。

2

准备参考资料

切分向量化

要让 Agent 能"翻书"，你得先把书准备好。过程是：

第一步：把文档切成小段。一整本书太长了，模型读不下。把它切成一段段（比如每段 500 字），这样检索时只拿最相关的几段。

第二步：给每段做"指纹"。计算机不能直接理解文字的含义，需要把文字变成一串数字（叫"向量"），语义相近的段落数字也相近。这样搜索时就能找到意思最接近的段落。

python
from openai import OpenAI

client = OpenAI()

def split_text(text, chunk_size=500, overlap=50):
    """把长文本切成带重叠的小段"""
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start = end - overlap
    return chunks

def get_embedding(text):
    """把文字变成向量（一串数字）"""
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    )
    return response.data[0].embedding

# 准备文档
doc = open("product_manual.txt", encoding="utf-8").read()
chunks = split_text(doc)
chunk_vectors = [(chunk, get_embedding(chunk)) for chunk in chunks]
print(f"文档切成了 {len(chunks)} 段")

检索并注入上下文

相似度上下文注入

用户提问时，先把问题也做成"指纹"，找出最相似的几段资料，塞给 Agent 一起读。

就像考前老师画重点——不给整本书，只给最相关的几页。Agent 读了这几页后回答，准确率大幅提升。

python
import numpy as np

def cosine_similarity(a, b):
    """计算两个向量的相似度（越接近1越相似）"""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

def search(query, chunk_vectors, top_k=3):
    """找出与问题最相关的 top_k 段文档"""
    query_vec = get_embedding(query)
    scored = []
    for chunk, vec in chunk_vectors:
        score = cosine_similarity(query_vec, vec)
        scored.append((score, chunk))
    scored.sort(reverse=True)
    return [chunk for _, chunk in scored[:top_k]]

def ask_with_context(question, chunk_vectors):
    """检索相关资料 → 注入上下文 → 让 Agent 回答"""
    relevant = search(question, chunk_vectors)
    context = "\n\n---\n\n".join(relevant)

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": (
                "根据以下参考资料回答用户问题。"
                "如果资料中没有相关信息，请明确说明。\n\n"
                f"参考资料：\n{context}"
            )},
            {"role": "user", "content": question}
        ]
    )
    return response.choices[0].message.content

完整 RAG Agent

完整流程

把前面的步骤串起来：加载文档 → 切分 → 向量化 → 用户提问时检索 → 注入上下文 → 回答。

这就是市面上大多数"知识库问答""智能客服""文档助手"产品的核心原理。

核心收获：RAG 不需要重新训练模型，只需要把你的文档"喂"给检索系统。Agent 回答前先翻资料，自然就不会胡编乱造了。
  • 向量数据库：示例用内存存储，生产环境用 Chroma、Pinecone、Weaviate 等专业向量数据库
  • 切分策略：按段落、按标题、按语义切分各有优劣，需要根据文档类型调整
  • 混合检索：向量检索 + 关键词检索结合，效果更好
  • 引用来源：让 Agent 在回答中注明"根据文档第X章"，增加可信度
