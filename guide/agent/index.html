理解 Agent：它到底是什么？

概念 LLM Tool Use

一个 AI Agent 的本质非常简单——它就是一个循环：

🧠
LLM
大语言模型思考
→
🔧
Tools
执行工具/函数
→
🔄
Loop
把结果喂回 LLM

普通 API 调用：你问一句，模型答一句，结束。
Agent 模式：模型自主决定调用哪些工具，拿到结果后继续思考，直到任务完成。

核心概念

  • LLM（大语言模型）：Agent 的"大脑"，负责理解任务、推理决策。如 GPT-4o、Claude 等。
  • Tool / Function Calling：模型不直接执行操作，而是输出"我想调用某个工具"的结构化指令，由你的代码执行后把结果返回。
  • Agent Loop：核心循环——发消息给 LLM → 检查是否要调用工具 → 执行工具 → 把结果追加到对话 → 重复，直到 LLM 给出最终回答。
2

环境准备

Python API Key 环境变量

只需要两样东西：Python 环境和一个 API Key。

bash 
# 1. 安装 OpenAI SDK
pip install openai

# 2. 设置 API Key（替换为你的 key）
export OPENAI_API_KEY="sk-..."

如果用国内平台（如 DeepSeek、智谱），只需改 base_url 和 key，代码完全一样。

API Key 是什么？

  • API Key 是你访问模型服务的"密钥"，按调用量计费。
  • 去 platform.openai.com 注册并创建 key。
  • 安全原则：永远不要把 key 硬编码到代码里，用环境变量或 .env 文件管理。

兼容其他模型

  • OpenAI SDK 支持任何兼容 OpenAI API 格式的服务。
  • 只需在初始化时传入 base_url 参数即可切换到 DeepSeek、Ollama 本地模型等。
3

第一次调用 LLM

Chat Completion Messages System Prompt

最基础的调用：发一组消息，拿到模型回复。

python 
from openai import OpenAI

client = OpenAI()  # 自动读取 OPENAI_API_KEY 环境变量

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "你是一个有帮助的助手。"},
        {"role": "user", "content": "Python 的 list 和 tuple 有什么区别？"}
    ]
)

print(response.choices[0].message.content)

消息角色（Roles）

  • system：设定 AI 的行为和人格，模型会始终遵循。
  • user：用户的输入。
  • assistant：模型的回复。多轮对话时需要把历史回复也放进来。
  • tool：工具的返回结果（下一步会用到）。

关键参数

  • model：选择模型，gpt-4o-mini 便宜快速适合入门，gpt-4o 更强。
  • temperature：0 = 确定性输出，1 = 更有创意。默认 1。

定义工具（Function Calling）

Function Calling JSON Schema Tools

告诉模型"你有哪些工具可以用"。模型会自主决定何时调用。

python 
import json

# 1. 定义工具的 schema（告诉模型工具长什么样）
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称，如 北京、上海"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

# 2. 实现工具函数（实际的执行逻辑）
def get_weather(city):
    # 实际项目中这里调用天气 API
    return json.dumps({"city": city, "temp": "22°C", "condition": "晴"})

Function Calling 工作流

  1. 你在请求中传入 tools 数组，描述每个工具的名称、用途、参数格式。
  2. 模型分析用户意图，如果需要调用工具，会返回 tool_calls 而非文本回复。
  3. 你的代码根据 tool_calls 执行对应函数，把结果作为 tool 消息追加到对话。
  4. 再次请求模型，模型拿到工具结果后给出最终回答。

JSON Schema

  • 工具参数用 JSON Schema 格式描述，模型据此生成正确的参数 JSON。
  • required 数组指定哪些参数是必填的。
5

构建 Agent 循环（核心）

Agent Loop tool_calls 多轮对话

这是 Agent 的核心——一个 while 循环，让模型自主调用工具直到完成任务。

python 
# 工具函数映射表
tool_functions = {
    "get_weather": get_weather,
}

def run_agent(user_input):
    messages = [
        {"role": "system", "content": "你是一个智能助手，可以查询天气。"},
        {"role": "user", "content": user_input}
    ]

    while True:
        # 1. 调用 LLM
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=messages,
            tools=tools
        )
        msg = response.choices[0].message

        # 2. 没有 tool_calls → 模型给出了最终回答，退出循环
        if not msg.tool_calls:
            return msg.content

        # 3. 有 tool_calls → 逐个执行工具
        messages.append(msg)  # 把 assistant 的 tool_calls 消息加入对话

        for call in msg.tool_calls:
            fn = tool_functions[call.function.name]
            args = json.loads(call.function.arguments)
            result = fn(**args)

            # 4. 把工具结果追加到对话
            messages.append({
                "role": "tool",
                "tool_call_id": call.id,
                "content": result
            })

        # 循环继续 → 模型拿到工具结果后再次思考
这就是全部了。 整个 Agent 的核心就是这 30 行代码。模型决定调不调工具、调哪个、调几次——你只需要执行并把结果喂回去。

循环的关键点

  • 退出条件：当模型不再返回 tool_calls 时，说明它已经有了最终答案。
  • 多次调用：模型可能在一次循环中调用多个工具，也可能循环多轮调用不同工具。
  • 对话历史：每次工具结果都追加到 messages，模型能看到完整上下文。

安全建议

  • 加一个最大循环次数（如 10 次），防止模型无限循环。
  • 验证模型返回的工具名称是否在白名单内。
  • 对工具参数做校验，不要盲目执行。

运行 & 扩展

完整代码 扩展方向

把前面的代码合在一起，加两行就能运行：

python 
from openai import OpenAI
import json

client = OpenAI()

tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称"}
                },
                "required": ["city"]
            }
        }
    }
]

def get_weather(city):
    return json.dumps({"city": city, "temp": "22°C", "condition": "晴"})

tool_functions = {"get_weather": get_weather}

def run_agent(user_input):
    messages = [
        {"role": "system", "content": "你是一个智能助手，可以查询天气。"},
        {"role": "user", "content": user_input}
    ]
    for _ in range(10):  # 最多循环 10 次
        response = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools
        )
        msg = response.choices[0].message
        if not msg.tool_calls:
            return msg.content
        messages.append(msg)
        for call in msg.tool_calls:
            fn = tool_functions[call.function.name]
            result = fn(**json.loads(call.function.arguments))
            messages.append({
                "role": "tool",
                "tool_call_id": call.id,
                "content": result
            })
    return "达到最大循环次数"

# 运行！
print(run_agent("北京和上海今天天气怎么样？"))
