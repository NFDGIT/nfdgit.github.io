Skill 是什么？

概念 SKILL.md 开放标准

你在前面学过 Tool（工具）——一个函数，模型调用一次就完成。Skill 是更高一层的抽象：它是一套完整的工作流指令，教 Agent 怎么完成一类多步骤任务。

🔧
Tool
单个函数调用
vs
📋
Skill
多步工作流指令

一个 Skill 就是一个目录，里面有一个 SKILL.md 文件：

text 
my-skill/
  └── SKILL.md          ← 只需要这一个文件
  └── scripts/           ← 可选：辅助脚本
        └── helper.py

Tool vs Skill

  • Tool：`get_weather(city)` — 一次调用返回结果，模型自主决定何时调用。
  • Skill：`code-review` — 一整套工作流（读代码 → 分析问题 → 写报告），按指令一步步执行。
  • Skill 可以内部使用多个 Tool，是更高层的能力封装。

支持 Skill 的工具

  • Cursor、Claude Code、OpenAI Codex、Gemini CLI、VS Code Copilot、Windsurf 等 16+ 工具
  • 都遵循同一个 SKILL.md 标准格式
  • 一个 Skill 写一次，在所有支持的工具中通用

编写 SKILL.md

YAML Frontmatter Markdown 指令

SKILL.md 由两部分组成：YAML 头部（声明元数据）+ Markdown 正文（教 Agent 怎么做）。

markdown 
---
name: deploy-checker
description: 部署前检查项目状态，确保没有遗漏
license: MIT
compatibility: Cursor, Claude Code
metadata:
  author: your-name
  version: "1.0"
---

# 部署前检查

当用户说"帮我检查一下能不能部署"时，执行以下步骤：

1. **检查 Git 状态**：运行 `git status`，确认没有未提交的更改
2. **运行测试**：执行项目的测试命令，确认全部通过
3. **检查依赖**：确认 package.json / requirements.txt 没有安全漏洞
4. **构建验证**：运行 build 命令，确认没有编译错误

## 输出格式

用清单形式展示每项检查结果：
- ✅ Git 状态干净
- ✅ 测试全部通过（42/42）
- ⚠️ 发现 1 个低风险依赖漏洞
- ✅ 构建成功

最后给出建议：可以部署 / 建议修复后再部署。

Frontmatter 字段

  • name（必填）：Skill 名称，kebab-case 格式
  • description（必填）：一句话描述，Agent 用它判断何时激活
  • license：开源协议，如 MIT、Apache-2.0
  • compatibility：兼容的工具列表
  • metadata：作者、版本、标签等扩展信息

Markdown 指令编写技巧

  • 步骤化：用编号列表，让 Agent 按顺序执行
  • 具体化：写清楚要运行什么命令、读什么文件、检查什么条件
  • 定义输出格式：告诉 Agent 结果怎么展示
  • 处理异常：说明出错时怎么办

在 Agent 中加载和使用 Skill

加载路径 System Prompt

在 Cursor / Claude Code 中：把 Skill 放到指定目录，工具自动加载。

bash 
# 项目级（仅当前项目可用）
mkdir -p .cursor/skills/deploy-checker
cp SKILL.md .cursor/skills/deploy-checker/

# 全局级（所有项目可用）
mkdir -p ~/.cursor/skills/deploy-checker
cp SKILL.md ~/.cursor/skills/deploy-checker/

# 使用：在聊天中输入
# "帮我检查一下能不能部署" → 自动匹配 deploy-checker Skill
# 或 @deploy-checker 手动指定

在自己写的 Agent 中：读取 SKILL.md 并注入 system prompt。

python 
from pathlib import Path
import yaml

def load_skill(skill_dir):
    """读取 SKILL.md，解析 frontmatter 和指令"""
    content = Path(skill_dir, "SKILL.md").read_text(encoding="utf-8")

    # 分离 YAML frontmatter 和 Markdown 正文
    parts = content.split("---", 2)
    meta = yaml.safe_load(parts[1])
    instructions = parts[2].strip()

    return {"meta": meta, "instructions": instructions}

def run_agent_with_skill(user_input, skill_dir):
    """加载 Skill 并注入 Agent"""
    skill = load_skill(skill_dir)

    messages = [
        {"role": "system", "content": (
            f"你是一个智能助手。\n\n"
            f"## 可用技能: {skill['meta']['name']}\n"
            f"{skill['meta']['description']}\n\n"
            f"当用户请求与此技能相关的任务时，按以下指令执行：\n\n"
            f"{skill['instructions']}"
        )},
        {"role": "user", "content": user_input}
    ]

    response = client.chat.completions.create(
        model="gpt-4o-mini", messages=messages
    )
    return response.choices[0].message.content

加载路径优先级

  • 项目级 .cursor/skills/ 优先于全局 ~/.cursor/skills/
  • 同名 Skill 项目级覆盖全局级
  • Claude Code 用 .claude/skills/，Codex 用 .codex/skills/

动态加载 vs 静态加载

  • 静态：所有 Skill 都放进 system prompt → 简单但占 token
  • 动态：先用 description 匹配用户意图，只加载相关 Skill → 省 token，适合 Skill 多的场景

实战：代码审查 Skill

完整示例 代码审查

写一个实用的代码审查 Skill，让 Agent 按统一标准审查代码：

markdown 
---
name: code-review
description: 对代码变更进行结构化审查，检查质量、安全和可维护性
license: MIT
metadata:
  author: your-name
  version: "1.0"
  tags: [code-quality, review]
---

# 代码审查技能

收到代码审查请求后，按以下维度逐一检查：

## 审查流程

1. **理解变更**：先读 diff 或文件，理解这次改动的目的
2. **正确性**：逻辑是否正确？边界条件是否处理？
3. **安全性**：有无注入、泄露、硬编码密钥等风险？
4. **可读性**：命名是否清晰？复杂逻辑有无注释？
5. **性能**：有无明显的性能问题（N+1 查询、不必要的循环）？
6. **测试**：变更是否有对应的测试覆盖？

## 输出格式

```
### 审查结果

**变更概述**：一句话描述这次改动

**问题清单**：
- 🔴 [严重] 问题描述 → 建议修复方式
- 🟡 [建议] 问题描述 → 改进方向
- 🟢 [良好] 值得肯定的做法

**总体评价**：通过 / 需要修改后再审
```

## 注意事项

- 只指出真正的问题，不要吹毛求疵
- 给出具体修复建议，不要只说"不好"
- 肯定好的做法，审查不只是找问题

在自己的 Agent 中使用：

python 
import glob

def load_all_skills(skills_dir=".cursor/skills"):
    """加载目录下所有 Skill"""
    skills = {}
    for skill_file in glob.glob(f"{skills_dir}/*/SKILL.md"):
        skill = load_skill(Path(skill_file).parent)
        skills[skill["meta"]["name"]] = skill
    return skills

def match_skill(user_input, skills):
    """简单关键词匹配，生产环境可用 embedding 相似度"""
    for name, skill in skills.items():
        desc = skill["meta"]["description"].lower()
        if any(kw in user_input.lower() for kw in desc.split()):
            return skill
    return None

# 使用
skills = load_all_skills()
skill = match_skill("帮我审查这段代码", skills)
if skill:
    result = run_agent_with_skill("帮我审查这段代码:\n...", skill)
    print(result)
核心收获：Skill 本质就是结构化的 prompt，用文件系统管理。写好 SKILL.md → 放到正确目录 → Agent 自动获得新能力。不需要改代码，不需要重启。
