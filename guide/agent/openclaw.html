1

OpenClaw 是什么？

框架 开源 本地部署

前面的教程你已经学会了手写 Agent。OpenClaw 把这些能力打包成了一个开箱即用的框架：

🖥️
Gateway
本地网关进程
⇄
💬
Channels
WhatsApp/Telegram/...
⇄
🧩
Skills
可扩展能力模块

核心特性：

  • 本地运行：数据不离开你的机器，隐私优先
  • 模型无关：支持 Claude、GPT-4o、DeepSeek、Ollama 本地模型等
  • 50+ 集成：消息平台、浏览器控制、文件系统、邮件、日历等
  • Skill 系统：社区 5000+ 技能可直接安装，也可自己写
  • MCP 原生支持：每个 Skill 都是 MCP Server

手写 Agent vs OpenClaw

  • 手写：你掌控一切——API 调用、工具执行、记忆管理全部自己实现。适合学习和定制化场景。
  • OpenClaw：框架替你处理网关、消息路由、持久化记忆、安全沙箱。你只需关注"让 Agent 做什么"（写 Skill）。
  • 建议：先理解原理（前面的教程），再用框架提效。

架构概览

  • Gateway：常驻后台的控制面，管理所有连接和任务调度。
  • Heartbeat：主动监控系统，无需用户触发即可检查待办、推送提醒。
  • ClawHub：技能市场，类似 npm/pip，一行命令安装社区技能。

安装与启动

安装 Node.js Dashboard

需要 Node.js 22+。一条命令安装：

bash 
# macOS / Linux 一键安装
curl -fsSL https://openclaw.ai/install.sh | bash

# 或者用 npm 全局安装
npm install -g openclaw@latest

安装完成后，运行引导配置：

bash 
# 1. 运行引导（配置模型 API Key、消息平台等）
openclaw onboard --install-daemon

# 2. 检查 Gateway 状态
openclaw gateway status

# 3. 打开 Dashboard — 浏览器访问 http://127.0.0.1:18789/
openclaw dashboard
最快上手路径：跳过所有平台配置，直接运行 openclaw dashboard，在浏览器里就能开始对话。

onboard 做了什么？

  • 创建配置目录 ~/.openclaw/
  • 引导你选择 AI 模型并输入 API Key（或配置本地 Ollama）
  • 可选：配置消息平台连接（WhatsApp、Telegram 等）
  • --install-daemon 将 Gateway 注册为系统服务，开机自启

Windows 用户

  • 推荐使用 WSL2，体验与 Linux 一致。
  • 或在 PowerShell 中运行 iwr -useb https://openclaw.ai/install.ps1 | iex

连接消息平台

WhatsApp Telegram Discord

OpenClaw 的强大之处在于它能连接你日常使用的消息平台，让 Agent 在聊天窗口中为你工作。

bash 
# 连接 WhatsApp（扫描二维码）
openclaw channel add whatsapp

# 连接 Telegram（输入 Bot Token）
openclaw channel add telegram

# 连接 Discord
openclaw channel add discord

# 查看所有已连接的平台
openclaw channel list

模型配置（如果 onboard 时跳过了）：

bash 
# 使用 OpenAI
openclaw model set openai --api-key "sk-..."

# 使用本地 Ollama 模型（完全离线）
openclaw model set ollama --model "llama3"

# 使用 Anthropic Claude
openclaw model set anthropic --api-key "sk-ant-..."

安全机制

  • 首次收到消息时需要在 Dashboard 中手动批准配对，防止陌生人使用你的 Agent。
  • 所有 API Key 加密存储在本地，不会上传到任何服务器。
  • 可以为不同联系人设置不同的权限级别。

支持的平台

  • WhatsApp、Telegram、Discord、Slack、Signal、Matrix
  • 邮件（IMAP/SMTP）
  • Web Dashboard（始终可用）

创建自定义 Skill

SKILL.md MCP ClawHub

Skill 是 OpenClaw 的扩展单元——一个文件夹 + 一个 SKILL.md 文件。

bash 
# 在技能目录创建新技能
mkdir -p ~/.openclaw/skills/daily-summary
cd ~/.openclaw/skills/daily-summary

创建 SKILL.md：

markdown 
---
name: daily-summary
version: "1.0"
description: 每天早上发送待办事项和天气摘要
triggers:
  - schedule: "0 8 * * *"   # 每天早上 8 点触发
  - command: /summary        # 手动触发命令
permissions:
  - network                  # 需要联网查天气
---

# 每日摘要技能

当触发时，执行以下步骤：

1. 获取用户所在城市的当天天气预报
2. 读取用户的待办事项列表（从 ~/todos.md）
3. 整合成一条简洁的早安消息，格式：

   **早安！今日概览：**
   - 天气：{天气信息}
   - 待办：{待办事项列表}
   - 建议：{根据天气和待办给出的建议}

保持消息简洁，不超过 200 字。

安装社区技能更简单：

bash 
# 从 ClawHub 安装社区技能
openclaw skill install web-search
openclaw skill install github-assistant
openclaw skill install smart-home

# 查看已安装技能
openclaw skill list

SKILL.md 结构

  • YAML Frontmatter：声明名称、版本、触发条件、权限需求。
  • Markdown 正文：用自然语言告诉 Agent 该怎么做——这就是 prompt engineering。
  • scripts/ 目录（可选）：放辅助脚本，Skill 中可以引用执行。

MCP 集成

  • OpenClaw 原生支持 MCP。每个 Skill 本质上就是一个 MCP Server。
  • 在 frontmatter 中声明 requires.bins 和 requires.env 即可引入外部 MCP 依赖。
  • 你在 MCP 教程中学到的所有内容，都可以直接用在 OpenClaw Skill 里。

安全提醒

  • 社区技能安装前务必查看源码——ClawHub 曾出现过恶意插件。
  • 敏感权限（filesystem、shell）的技能会在安装时提示确认。
