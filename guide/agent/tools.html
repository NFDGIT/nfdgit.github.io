定义多个工具

Tools 数组 JSON Schema

tools 是一个数组，你可以放任意多个工具定义。模型会根据用户意图自行选择调用哪个。

python 
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称"}
                },
                "required": ["city"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "search_web",
            "description": "搜索互联网获取最新信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "搜索关键词"}
                },
                "required": ["query"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "run_python",
            "description": "执行 Python 代码并返回结果",
            "parameters": {
                "type": "object",
                "properties": {
                    "code": {"type": "string", "description": "要执行的 Python 代码"}
                },
                "required": ["code"]
            }
        }
    }
]

工具设计原则

  • 名称清晰：用动词开头，如 get_weather、search_web、run_python。
  • 描述准确：模型靠 description 决定何时调用，描述越精确误调越少。
  • 参数最少：只暴露必要参数，减少模型出错概率。
  • 返回 JSON 字符串：工具函数统一返回 json.dumps() 格式。

实现工具函数

函数实现 subprocess

每个工具 schema 对应一个实际的 Python 函数。

python 
import json
import subprocess

def get_weather(city):
    # 实际项目中调用天气 API
    return json.dumps({"city": city, "temp": "22°C", "condition": "晴"})

def search_web(query):
    # 实际项目中调用搜索 API（Google、Bing 等）
    return json.dumps({
        "results": [
            {"title": f"关于 {query} 的结果", "snippet": "这是搜索摘要..."}
        ]
    })

def run_python(code):
    try:
        result = subprocess.run(
            ["python3", "-c", code],
            capture_output=True, text=True, timeout=10
        )
        output = result.stdout or result.stderr
        return json.dumps({"output": output.strip()})
    except subprocess.TimeoutExpired:
        return json.dumps({"error": "执行超时（10秒限制）"})

# 工具映射表——key 必须与 schema 中的 name 完全一致
tool_functions = {
    "get_weather": get_weather,
    "search_web": search_web,
    "run_python": run_python,
}
安全提示：run_python 这类执行代码的工具风险极高。生产环境中应使用沙箱（Docker 容器等）隔离执行。

处理并行工具调用

parallel_tool_calls 多工具

模型可能在一次回复中调用多个工具（如同时查北京和上海天气）。你的循环已经能处理这种情况——msg.tool_calls 是一个列表，逐个执行即可。

python 
for call in msg.tool_calls:
    name = call.function.name

    # 安全校验：只执行白名单内的工具
    if name not in tool_functions:
        result = json.dumps({"error": f"未知工具: {name}"})
    else:
        args = json.loads(call.function.arguments)
        result = tool_functions[name](**args)

    messages.append({
        "role": "tool",
        "tool_call_id": call.id,
        "content": result
    })

并行调用细节

  • 模型默认开启并行调用。如需禁用，在请求中设置 parallel_tool_calls=False。
  • 每个 tool_call 有唯一的 id，回复时必须通过 tool_call_id 匹配。
  • 如果某个工具执行失败，返回错误信息而非抛异常——让模型决定如何处理。

工具校验清单

  • 工具名称是否在白名单内？
  • 参数类型是否符合预期？
  • 敏感操作（删除文件、执行代码）是否需要二次确认？

完整多工具 Agent

完整代码

把以上步骤合起来，一个支持多工具的 Agent 就完成了：

python 
from openai import OpenAI
import json, subprocess

client = OpenAI()

tools = [
    {"type": "function", "function": {
        "name": "get_weather",
        "description": "获取指定城市的当前天气",
        "parameters": {"type": "object", "properties": {
            "city": {"type": "string"}}, "required": ["city"]}
    }},
    {"type": "function", "function": {
        "name": "search_web",
        "description": "搜索互联网获取最新信息",
        "parameters": {"type": "object", "properties": {
            "query": {"type": "string"}}, "required": ["query"]}
    }},
    {"type": "function", "function": {
        "name": "run_python",
        "description": "执行 Python 代码并返回结果",
        "parameters": {"type": "object", "properties": {
            "code": {"type": "string"}}, "required": ["code"]}
    }},
]

def get_weather(city):
    return json.dumps({"city": city, "temp": "22°C", "condition": "晴"})

def search_web(query):
    return json.dumps({"results": [{"title": f"{query}", "snippet": "搜索结果摘要"}]})

def run_python(code):
    try:
        r = subprocess.run(["python3", "-c", code],
                           capture_output=True, text=True, timeout=10)
        return json.dumps({"output": (r.stdout or r.stderr).strip()})
    except subprocess.TimeoutExpired:
        return json.dumps({"error": "超时"})

tool_functions = {"get_weather": get_weather, "search_web": search_web, "run_python": run_python}

def run_agent(user_input):
    messages = [
        {"role": "system", "content": "你是全能助手，可以查天气、搜索网页、执行代码。"},
        {"role": "user", "content": user_input}
    ]
    for _ in range(10):
        resp = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=tools)
        msg = resp.choices[0].message
        if not msg.tool_calls:
            return msg.content
        messages.append(msg)
        for call in msg.tool_calls:
            fn = tool_functions.get(call.function.name)
            if fn:
                result = fn(**json.loads(call.function.arguments))
            else:
                result = json.dumps({"error": "未知工具"})
            messages.append({"role": "tool", "tool_call_id": call.id, "content": result})
    return "达到最大循环次数"

print(run_agent("帮我查一下北京天气，再算一下 2 的 10 次方"))
