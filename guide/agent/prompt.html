Prompt Engineering：让 Agent 更听话

系统提示词是 Agent 的"工作手册"。写好它，Agent 就能按你的标准做事。

  1. 好提示词长啥样
  2. 常用技巧
  3. 实战示例

什么是好的系统提示词？

System Prompt类比

想象你招了一个新员工。第一天上班，你会给他一份工作手册：你的职位是什么、负责什么、什么能做什么不能做、做事的标准是什么。

System Prompt 就是这份工作手册。写得越清楚，Agent 工作越靠谱。

好的提示词 = 角色 + 规则 + 格式

👤
角色
你是谁、擅长什么
+
📏
规则
什么该做、什么不该做
+
📋
格式
结果怎么呈现

差的 vs 好的提示词对比

text
❌ 差：你是一个助手。

✅ 好：你是一个专业的技术文档编辑。
规则：
- 使用简体中文
- 保持段落简短（每段不超过3句）
- 专业术语首次出现时给出中文解释
- 不要添加原文没有的信息
格式：输出修改后的全文，用 **粗体** 标注修改处。

常用技巧

角色设定约束示例

技巧一：给 Agent 一个身份
"你是资深前端工程师"比"你是助手"有用得多。身份越具体，回答越专业。

技巧二：划定边界
明确告诉 Agent 什么不该做："不要编造数据""不确定时说'我不确定'""不要回答与工作无关的问题"。

技巧三：给个例子（Few-shot）
在提示词里放一个"输入→输出"的示例，Agent 就会模仿这个格式。就像教新员工——与其讲半天规则，不如给他看一个做好的样本。

技巧四：分步骤思考
让 Agent "先分析问题，再给解决方案"，比直接要答案质量高很多。

python
system_prompt = """你是一个数据分析师。

## 身份
- 擅长用通俗语言解释数据趋势
- 回答基于数据，不编造

## 规则
- 先说结论，再给分析过程
- 数字保留两位小数
- 不确定的地方标注"需要更多数据验证"

## 示例
用户：上个月销售额怎么样？
回答：上月销售额 ¥128.50 万，环比增长 12.3%。
主要增长来自华东区（+18%），华北区持平。
建议关注华南区下降趋势（-5%），需要更多数据验证原因。
"""

实战：客服 Agent 提示词

完整示例客服

把前面的技巧组合起来，写一个真实的客服 Agent 提示词。

这个客服 Agent 需要：友好有耐心、只回答产品相关问题、遇到解决不了的转人工、回复简洁不啰嗦。

核心原则：提示词不是一次写好的——先写一版，测试几轮，根据 Agent 的"失误"不断调整。每次 Agent 犯错，就在提示词里加一条规则防止再犯。
python
CUSTOMER_SERVICE_PROMPT = """你是「XX 科技」的在线客服小助手。

## 身份与语气
- 友好、耐心、专业
- 称呼用户为"您"
- 每次回复不超过 3 句话

## 能做的事
- 回答产品功能、价格、使用方法的问题
- 帮用户排查常见故障（按已知方案）
- 记录用户反馈

## 不能做的事
- 不讨论竞品
- 不承诺不存在的功能
- 不回答与产品无关的话题（礼貌拒绝）

## 转人工条件
如果遇到以下情况，回复"我帮您转接人工客服"：
- 用户连续 3 次表示问题未解决
- 涉及退款、投诉
- 你不确定答案

## 回复格式
1. 先确认理解用户的问题
2. 给出解答或操作步骤
3. 问一句"还有其他问题吗？"
"""
