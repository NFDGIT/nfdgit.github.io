为 Agent 加入记忆

让 Agent 记住之前的对话，跨会话保持上下文。

  1. 理解记忆
  2. 文件持久化
  3. 滑动窗口

1

理解"记忆"问题

Context Window 无状态

LLM 本身是无状态的——每次调用都是全新的。所谓"记忆"，就是你把历史对话放进 messages 数组再发给模型。

问题在于：模型有 上下文窗口限制（如 128K tokens），对话太长会溢出。需要策略管理。

上下文窗口

  • 每个模型有 token 上限：GPT-4o 为 128K，GPT-4o-mini 也是 128K。
  • messages 中所有内容（system + 历史 + 工具结果）都占用 token。
  • 超出上限会报错，需要在发送前裁剪或压缩。

常见记忆策略

  • 完整记忆：保存全部历史，简单但会爆。
  • 滑动窗口：只保留最近 N 轮。
  • 摘要记忆：定期让模型总结历史，用摘要替换原文。
2

文件持久化：最简单的记忆

JSON 文件 持久化

把对话历史存到 JSON 文件，下次启动时读取恢复。

python 
import json
from pathlib import Path

MEMORY_FILE = Path("agent_memory.json")

def load_memory():
    """加载历史对话，文件不存在则返回空列表"""
    if MEMORY_FILE.exists():
        return json.loads(MEMORY_FILE.read_text(encoding="utf-8"))
    return []

def save_memory(messages):
    """保存对话到文件（过滤掉 system 消息）"""
    to_save = [m for m in messages if m.get("role") != "system"]
    MEMORY_FILE.write_text(
        json.dumps(to_save, ensure_ascii=False, indent=2),
        encoding="utf-8"
    )

def run_agent_with_memory(user_input):
    history = load_memory()
    messages = [
        {"role": "system", "content": "你是一个有记忆的助手。"},
        *history,
        {"role": "user", "content": user_input}
    ]

    response = client.chat.completions.create(
        model="gpt-4o-mini", messages=messages
    )
    reply = response.choices[0].message.content

    # 把本轮对话追加到历史并保存
    history.append({"role": "user", "content": user_input})
    history.append({"role": "assistant", "content": reply})
    save_memory(history)

    return reply

滑动窗口：控制记忆长度

滑动窗口 Token 管理

历史太长时只保留最近 N 轮对话，防止超出上下文窗口。

python 
MAX_HISTORY_PAIRS = 20  # 保留最近 20 轮对话（40 条消息）

def trim_history(messages):
    """只保留最近 N 轮 user+assistant 对"""
    non_system = [m for m in messages if m["role"] != "system"]
    if len(non_system) > MAX_HISTORY_PAIRS * 2:
        non_system = non_system[-(MAX_HISTORY_PAIRS * 2):]
    return non_system

# 在 run_agent_with_memory 中使用：
# history = trim_history(load_memory())

为什么不直接数 token？

  • 精确计数需要 tiktoken 库，按轮数裁剪更简单。
  • 如需精确控制，可以用 tiktoken.encoding_for_model("gpt-4o-mini") 计算每条消息的 token 数。

更高级的裁剪

  • 按 token 总量裁剪：从最旧的消息开始删，直到总 token 低于阈值。
  • 保留重要消息：标记"关键对话"不被裁剪。

摘要记忆：压缩历史

摘要 长期记忆

让模型把旧对话总结成一段摘要，用摘要替换原文，既保留关键信息又省 token。

python 
def summarize_history(messages):
    """让模型总结旧对话"""
    conversation_text = "\n".join(
        f'{m["role"]}: {m["content"]}' for m in messages
    )
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{
            "role": "user",
            "content": f"请用 3-5 句话总结以下对话的关键信息：\n\n{conversation_text}"
        }]
    )
    return response.choices[0].message.content

def smart_memory(history, threshold=30):
    """历史超过阈值时，前半部分压缩为摘要"""
    if len(history) <= threshold:
        return history

    # 前半部分总结，后半部分保留原文
    mid = len(history) // 2
    summary = summarize_history(history[:mid])
    return [
        {"role": "assistant", "content": f"[历史摘要] {summary}"},
        *history[mid:]
    ]
组合使用：实际项目中常常同时使用文件持久化 + 滑动窗口 + 定期摘要，三者配合效果最佳。
